Phase 1 — Input & Initialization
Step 1

📂 Upload Marksheet PDF

Dept-Admin uploads scanned marksheet PDF via React frontend.
API Endpoint: POST /upload-marksheet with file, semester, user info.

Step 2

💾 Save Temp File & Load Config

Backend (FastAPI) saves PDF to temp_uploads/ with unique UUID filename.
Semester label & expected names (if any) are stored.

Step 3

🗄️ Load Ground-Truth Names from MySQL

Query student_name table for all registered student names.
Merge with any extra names from frontend & Excel fallbacks.
This forms the KNOWN_NAMES list for fuzzy matching.

Phase 2 — Image Preprocessing (per page)
Step 4

📄 PDF → High-Resolution Images

Convert each PDF page to image at 300 DPI using pdf2image.
Each page processed independently in a loop.

Step 5

🔲 Grayscale & Adaptive Binarization

Convert to grayscale → Apply Adaptive Gaussian Threshold (block=21, C=10)
to produce inverted binary image (white lines on black background).

Step 6

🔄 Deskew Correction

Detect skew angle via cv2.minAreaRect() on binary foreground pixels.
If |angle| > 0.1°, rotate image using affine warp to straighten.
Re-threshold after deskew for clean lines.

Phase 3 — Vision-Based Grid Detection
Step 7

📐 Canny Edge Detection

Apply cv2.Canny(gray, 50, 150) to detect edges.
Then use morphological operations to isolate
horizontal lines (wide kernel) and vertical lines (tall kernel).

Step 8

📊 Extract Row & Column Boundaries

Project line images along axes → find peaks → cluster nearby peaks.
Produces y_bounds[] (row positions) and x_bounds[] (column positions).

Decision

Sufficient Grid Structure?

Check: x_bounds ≥ 3 AND y_bounds ≥ 10?

✗ NO — Sparse Grid

🔁 Fallback: Threshold-based Detection

Re-run detect_lines() on adaptive binary image.
Merge results if they yield more boundaries.

Move to next page

✗ NO

✅ Continue

✓ YES — Grid OK

Phase 4 — Adaptive Column Identification
Step 9

📏 Identify Name & Result Columns

Find the widest gap in the first 5 column boundaries → that's the Name Column.
Result area: rightmost ~35% of the page.
Handles variable left-margin layouts (extra thin border columns).

Phase 5 — Row-by-Row Student Detection & Extraction
Step 10

🔁 Iterate Row Sub-Regions (while loop)

For each row boundary pair, crop the Name column ROI.

Step 11

🔍 OCR + Upscale Name Region

If ROI height < 200px → bicubic upscale to 200px.
Run Tesseract OCR (PSM 6) on the name cell.
Scan ahead up to 5 sub-rows to find a valid name.

Step 12

🧠 Fuzzy Match Against KNOWN_NAMES

Strategy 1: difflib.get_close_matches() with cutoff ≥ 0.55
Strategy 2: Substring containment for long names (>10 chars)
Noise filtering: reject headers (UNIVERSITY, SEMESTER, MARKS, etc.)

Decision

Valid Student Name Found?

Is the match score ≥ cutoff AND not already seen?

✗ NO

⏭️ Skip Row, i += 1

Continue to next sub-row

✓ YES

📌 Lock Student Block

Detect block end via next name / Sr No

Phase 6 — Multi-Mode GPA Extraction
Step 13

🅰️ Mode A: Composite Block OCR

Crop entire right portion (65%→edge) of student block.
Run OCR with PSM 3 & PSM 6. Parse for last valid float in 4.0–10.0 range.

Decision

GPA Extracted?

Was a valid GPA value (4.0–10.0) detected OR FAIL status found?

✗ NO — Fallback
Step 14

🅱️ Mode B: Row-by-Row Fallback

Scan each sub-row individually with PSM 6 & PSM 11.
Parse each for GPA/FAIL status.

GPA Extracted Now?

✗ NO

🅲 Mode C: Last-Column Scan

Scan rightmost 2 columns with PSM 6, 7, 11

✓ YES

✅ GPA Locked

Proceed with extracted value

✓ YES

✅ GPA Locked

Proceed with extracted GPA & Status

Step 15

📋 Status Detection (PASS / FAIL)

Fail-first logic: regex for \bFAIL\b, \bATKT\b, NULL, (F).
If none found → Status = PASS. GPA set to 0.0 on FAIL.

Step 16

📝 Append Result Tuple

Add (student_name, gpa, status) to results list.
Mark name as seen to prevent duplicates. Move to next block.

Decision

More Rows / Pages Remaining?

Loop back until all sub-rows and all pages are processed.

✓ YES

🔁 Loop Back to Step 10

Process next row / next page

✗ NO — All Done

✅ Extraction Complete

Return full results list to API

Phase 7 — Output Formatting & Display
Step 17

🧹 Format Results for Frontend

Build JSON response: assign GPA to target semester, set status to PASS/FAIL,
generate roll numbers, student emails. Identify undetected students
(in DB but not found by OCR) for manual entry notification.

Step 18

📊 Display Results on React Frontend

JSON response rendered as an interactive student results table.
Dept-Admin reviews, edits (if needed), and confirms the data.
Undetected students flagged for manual GPA entry.

Decision

Admin Triggers Email Dispatch?

Does Dept-Admin click "Send Results via Email"?

✓ YES
Step 19

📧 SMTP Email Dispatch

Iterate over students → compose personalized email with name, GPA, avg pointer.
Send via Gmail SMTP (POST /send-results).
Log success/failure count.

✗ NO

📋 Results Displayed Only

Data available on dashboard for review

Step 20

📝 Activity Logging

Log the action (Result Generation / Bulk Mailing) with timestamp,
user name, and details for Super-Admin audit trail. Cleanup temp files.
